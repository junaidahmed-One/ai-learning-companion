# AI Learning Companion

<img width="1920" height="1100" alt="image" src="https://github.com/user-attachments/assets/09387d20-ab03-4239-84df-7488dd233600" />


Converso is a real-time, voice-first AI teaching platform built with Next.js. It allows users to create personalized learning "Companions"—AI tutors tailored to specific subjects and topics. Engage in natural, interactive voice conversations to master new concepts in a fun and effective way.

## Features

-   **Custom Companion Creation:** Build AI tutors from scratch, defining their name, subject, topic, voice, and conversational style.
-   **Interactive Voice Sessions:** Participate in real-time, voice-activated learning lessons powered by the Vapi AI platform.
-   **Dynamic Audio Visualization:** A Lottie animation provides visual feedback when the AI companion is speaking.
-   **Companion Library:** Browse a public library of companions. Search by topic or filter by subject to find the perfect tutor.
-   **User Authentication:** Secure sign-up, sign-in, and profile management handled by Clerk.
-   **Personalized Profiles:** Track your lesson history and manage the companions you've created.
-   **Subscription Tiers:** Companion creation limits are based on user subscription plans, managed through Clerk's role-based access control.

## Tech Stack

-   **Framework:** Next.js (App Router)
-   **Authentication:** Clerk
-   **Database:** Supabase
-   **Voice AI:** Vapi
-   **Styling:** Tailwind CSS
-   **UI Components:** Shadcn/ui
-   **Error Monitoring:** Sentry

## Getting Started

To run this project locally, follow the steps below.

### Prerequisites

-   Node.js (v20 or later)
-   npm, yarn, or pnpm
-   Git

### 1. Clone the Repository

```bash
git clone https://github.com/junaidahmed-one/ai-learning-companion.git
cd ai-learning-companion
```

### 2. Install Dependencies

```bash
npm install
# or
yarn install
# or
pnpm install
```

### 3. Set Up Environment Variables

Create a `.env.local` file in the root of the project and add the following environment variables. You will need to create accounts with Clerk, Supabase, Vapi, and Sentry to get the required keys.

```env
# Clerk
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=
CLERK_SECRET_KEY=
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in

# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=

# Vapi
NEXT_PUBLIC_VAPI_WEB_TOKEN=

# Sentry (Optional)
SENTRY_DSN=
SENTRY_ORG=
SENTRY_PROJECT=
SENTRY_AUTH_TOKEN=
```

### 4. Set Up Supabase Database

1.  Navigate to your Supabase project dashboard.
2.  Go to the **SQL Editor** and run the following queries to create the necessary tables.

    **Create the `companions` table:**
    ```sql
    CREATE TABLE public.companions (
        id uuid DEFAULT gen_random_uuid() NOT NULL,
        created_at timestamp with time zone DEFAULT now() NOT NULL,
        name text,
        subject text,
        topic text,
        voice text,
        style text,
        duration integer,
        author uuid REFERENCES auth.users(id)
    );
    ALTER TABLE public.companions ENABLE ROW LEVEL SECURITY;
    ```
    **Create the `session_history` table:**
    ```sql
    CREATE TABLE public.session_history (
        id bigint GENERATED BY DEFAULT AS IDENTITY,
        created_at timestamp with time zone DEFAULT now() NOT NULL,
        user_id uuid REFERENCES auth.users(id),
        companion_id uuid REFERENCES public.companions(id)
    );
    ALTER TABLE public.session_history ENABLE ROW LEVEL SECURITY;
    ```
3.  Set up Row Level Security (RLS) policies according to your application's needs. For a basic setup, you can allow authenticated users to read all data and insert/update their own data.

### 5. Run the Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result. You can start by creating an account, building a new companion, and launching a lesson.
